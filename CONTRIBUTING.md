## Guidelines

### Adding a Resource

1. **Ensure quality** - The resource should be high-quality and actively maintained
2. **Check for duplicates** - Verify it's not already in the list
3. **Alphabetical order** - Add resources in alphabetical order within their category
4. **Correct format** - Use the format: `[Resource Name](link) - Brief description`
5. **Brief descriptions** - Keep descriptions under 100 characters

### Quality Standards

‚úÖ **Do Add:**
- Free resources or those with free tiers
- Actively maintained (updated within last 2 years)
- Relevant to indie game development
- English language or has English documentation
- Resources you've personally used or can vouch for

‚ùå **Don't Add:**
- Affiliate links
- Self-promotion without disclosure
- Paid-only resources without significant value
- Abandoned projects (no updates in 2+ years)
- Resources with significant security issues

### Format Examples

**For tables:**
```markdown
| Tool Name | Category | Price | [toolname.com](https://toolname.com) |
```

**For lists:**
```markdown
- **Tool Name** - Brief description - [toolname.com](https://toolname.com)
```

**For simple entries:**
```markdown
- [Resource Name](link) - Brief description
```

## Pull Request Process

1. **Fork the repository**
   ```bash
   gh repo fork xeldar/awesome-indie-gamedev
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b add-resource-name
   ```

3. **Make your changes**
   - Follow the formatting guidelines above
   - Test all links work correctly
   
4. **Commit with a clear message**
   ```bash
   git commit -m "Add [Resource Name] to [Category]"
   ```

5. **Submit PR with description**
   - Explain what resource you're adding and why
   - Include any relevant context

## Reporting Issues

### Broken Links
If you find a broken link, please [open an issue](../../issues/new?template=report-broken.md) with:
- The resource name
- The current (broken) URL
- A working URL if you know one

### Outdated Resources
For outdated or deprecated resources:
- Check if a replacement exists
- Open an issue or PR to remove/update

### Misplaced Categories
If a resource seems to be in the wrong category:
- Open an issue explaining the suggested move
- Or submit a PR with the change

### Typos or Errors
- Small fixes can be submitted directly as PRs
- No need to open an issue first for typos
